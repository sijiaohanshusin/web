# HEU ESTA 官网维护手册

> 写给每一届接手网站的同学。看完这份文档，你应该能独立完成日常维护和二次开发。
> 遇到问题先看「故障排查」一节；再解决不了，把报错信息发给 AI 助手（Cursor / DeepSeek 等），效率最高。

## 1. 整体架构

```text
用户浏览器
   │
   ▼
CDN（缓存静态资源，减轻服务器压力）
   │
   ▼
阿里云服务器（Ubuntu 24.04，2核/1.6G + 2G swap）
   │
   ├─ nginx（宿主机，443 端口，SSL 证书 Let's Encrypt）
   │    ├─ /static/  → /srv/heuesta/static/   （CSS/JS/图片，一年长缓存）
   │    ├─ /media/   → /srv/heuesta/media/    （后台上传的轮播图、头像）
   │    ├─ /media/resources/ → 403            （资料文件禁止直链）
   │    ├─ /protected/ → internal             （Django 鉴权后由 nginx 发文件）
   │    ├─ /learn/   → /srv/heuesta/site/learn/（电子学学习中心，纯静态）
   │    └─ /         → 127.0.0.1:8001         （反向代理到 Django）
   │
   ├─ Docker: heuesta-app（Django + gunicorn，2 worker）
   ├─ Docker: heuesta-db （PostgreSQL 16）
   └─ Docker: hbbs/hbbr  （RustDesk 中继，与网站无关，别动）
```

关键路径速查：

| 位置 | 内容 |
| --- | --- |
| `/opt/heuesta/web` | 代码仓库（git checkout） |
| `/opt/heuesta/.env` | 生产密钥（SECRET_KEY、数据库密码），**不在 git 里，删了就没了** |
| `/srv/heuesta/static` | collectstatic 输出 |
| `/srv/heuesta/media` | 用户上传文件 |
| `/srv/heuesta/pgdata` | 数据库数据 |
| `/srv/heuesta/backups` | 每日自动备份（保留 7 天） |
| `/etc/nginx/sites-available/heuesta.cn` | nginx 站点配置 |

## 2. 本地开发环境

见 README「本地开发」。要点：

- 开发用 SQLite（`app/db.sqlite3`），不用装 Postgres
- `python manage.py runserver` 默认加载 `config.settings.dev`
- 前端零构建链：改 `app/static/` 下的 CSS/JS 刷新即见效
- 改了模型（models.py）之后：`python manage.py makemigrations && python manage.py migrate`
- 提交前跑测试：`python manage.py test accounts files`

## 3. 日常维护任务

日常操作首选**管理驾驶舱** `https://heuesta.cn/dashboard/`（干事及以上可登录；导航栏和个人中心都有入口）。它包含：

- **数据总览**：会员/资料/下载统计卡片，注册趋势、年级分布、资料分类、下载 Top10、B 站粉丝走势图表（B 站快照每天打开总览时自动记录）
- **会员管理**：待审核列表、搜索筛选、批量「通过审核」；「拒绝并删除」「设/撤干事」仅管理员可见
- **资料管理**：全量资料列表、页内上传、删除（干事只能删自己传的）
- **站点设置**（仅管理员）：招新视频 BV 号、QQ 群号等配置 + 首页轮播图上传/排序/启停/删除

`https://heuesta.cn/admin/`（高级后台，simpleui 主题）用于底层数据管理，日常用不到。

### 3.1 审核新会员

驾驶舱 → 会员管理 → 「待审核」标签（侧边栏有红色角标提醒）→ 勾选 → 「✓ 通过审核」。

### 3.2 设置干事 / 管理员

- 干事：驾驶舱 → 会员管理 → 「全部成员」→ 勾选 → 「设为干事」（管理员操作）
- 管理员（能进驾驶舱站点设置和高级后台）：高级后台 → 成员 → 编辑，加入 `管理员` 组并勾选「工作人员状态」
- 完全管理权：再勾选「超级用户状态」（谨慎）

### 3.3 换招新视频 / 改 QQ 群号

驾驶舱 → 站点设置 → 基本配置，改完保存即生效（首页缓存最多 1 小时后刷新）。

### 3.4 更新首页轮播图（科协合影等）

驾驶舱 → 站点设置 → 添加轮播图（建议宽 1600px、500KB 以内）。
只要有启用的轮播图，首页走廊就用后台上传的；一张没有时才显示仓库自带的兜底图 + 留白占位格。

### 3.5 上传资料

驾驶舱 → 资料管理 → 「+ 上传新资料」；前台资料站页面也有上传按钮。「仅会员」的资料非会员无法下载（鉴权在 Django，文件传输在 nginx，大文件也不占应用内存）。

## 4. 发布新版本

```bash
# 本地
git add -A && git commit -m "更新说明" && git push origin main

# 服务器（ssh aliyun）
sudo /opt/heuesta/web/ops/deploy.sh
```

deploy.sh 是幂等的，失败了修复后再跑一次即可。它不会动数据库数据和上传文件。

## 5. 备份与恢复

- 自动备份：`heuesta-backup.timer` 每天 03:30 备份数据库 + media 到 `/srv/heuesta/backups`，保留 7 天
- 手动备份：`sudo /opt/heuesta/web/ops/backup.sh`
- **建议每学期把备份拷一份到本地**：`scp aliyun:/srv/heuesta/backups/db-最新.sql.gz .`

恢复数据库（危险操作，先确认备份文件没问题）：

```bash
cd /opt/heuesta/web
gunzip -c /srv/heuesta/backups/db-XXXX.sql.gz | \
  docker compose -f ops/docker-compose.yml --env-file /opt/heuesta/.env \
  exec -T db psql -U heuesta -d heuesta
```

恢复上传文件：`tar -xzf media-XXXX.tar.gz -C /srv/heuesta`

## 6. 如何做二次开发

### 加一个新页面

1. `app/core/views.py` 加视图函数
2. `app/core/urls.py` 加路由
3. `app/templates/core/` 加模板（`{% extends "base.html" %}` 即自动获得导航栏页脚）
4. 导航栏入口在 `app/templates/base.html`

### 加一个新模块（比如论坛）

1. `python manage.py startapp forum`
2. `config/settings/base.py` 的 `INSTALLED_APPS` 里注册
3. `config/urls.py` 挂路由：`path("forum/", include("forum.urls"))`
4. 权限直接复用 `accounts.roles` 里的 `is_member` / `is_officer`
5. 用户外键写 `settings.AUTH_USER_MODEL`

会员系统、角色组、登录鉴权都是现成的，论坛只需要写帖子/回复的模型和页面。

### 更新学习中心

`learn/electronics/` 是纯静态文件，直接改 HTML，push 后 deploy 即可。新增章节记得同步 `manifest.json` 与 `index.html`。

## 7. 故障排查

| 症状 | 排查 |
| --- | --- |
| 网站 502 | `docker ps` 看 app 容器是否在跑；`docker compose ... logs app` 看报错 |
| 部署后页面没变 | 浏览器强刷（Ctrl+F5）；去腾讯 EdgeOne 控制台「站点加速 → 缓存刷新」刷新 heuesta.cn；确认 deploy.sh 跑完没报错。动态页面源站已发送 `Cache-Control: private, no-cache`，正常情况下 CDN 不会再缓存 HTML，只缓存 /static/ /media/ /learn/ |
| 上传文件失败 | 看 nginx `client_max_body_size`（现 200m）与磁盘空间 `df -h` |
| B 站数据不显示 | 正常降级，B 站接口偶尔抽风，1 小时缓存过期后自动恢复；不影响其他内容 |
| 内存吃紧 | `free -h`；app 容器限 450m、db 限 300m；已配 2G swap 兜底 |
| 证书过期 | certbot 自动续期：`systemctl list-timers | grep certbot`；手动 `certbot renew` |
| 忘了管理员密码 | `docker compose ... exec app python manage.py changepassword <用户名>` |

## 8. 论坛（bbs.heuesta.cn，NodeBB）

论坛是独立容器，和主站互不影响；账号与主站互通（主站登录后打开论坛即自动登录，无需注册）。

### 8.1 日常管理

- 论坛管理后台：`https://bbs.heuesta.cn/admin`（NodeBB 自带 ACP），管理员账号见交接密码册
- 设版主：ACP → 管理 → 版块 → 选版块 → 权限，把干事账号加为 moderator
- 账号统一由主站管理：论坛本地注册已关闭，用户名/密码修改都在主站进行

### 8.2 运维命令（服务器上）

```bash
# 重启论坛（不影响主站）
docker compose -f /opt/heuesta/web/ops/forum/docker-compose.yml --env-file /opt/heuesta/forum.env restart

# 看论坛日志
docker compose -f /opt/heuesta/web/ops/forum/docker-compose.yml --env-file /opt/heuesta/forum.env logs -f forum

# 升级 NodeBB：改 ops/forum/Dockerfile 里的 NODEBB_VERSION，然后
sudo /opt/heuesta/web/ops/forum/deploy-forum.sh
```

- 账号互通密钥：`/opt/heuesta/forum.env` 的 `NODEBB_JWT_SECRET` 必须与 `/opt/heuesta/.env` 中同名项一致，
  且与论坛 ACP → 插件 → Session Sharing 里的 Secret 一致，改任何一处都要三处同步
- 论坛数据：数据库 `nodebb`（与主站同一 postgres 实例、不同库）、附件 `/srv/heuesta/forum/uploads`，
  每日备份已包含（`forum-db-*.sql.gz` 与 `forum-uploads-*.tar.gz`）

### 8.3 论坛皮肤与版块管理（科协风格化）

皮肤与装饰全部由幂等脚本管理，位于 `ops/forum/`：

| 脚本 | 管什么 |
| --- | --- |
| `theme-v2.js` | 全站皮肤 CSS（PCB 蓝背景 + 米黄纸张 + 铜色装饰）+ 帖子页装饰 JS（楼层号、左侧作者栏、首页横幅、QQ 公告条） |
| `categories-v2.js` | 版块体系：6 大分区 14 子版块的名称/图标/配色/排序 |
| `seed-v2.js` | 各版块置顶导航/规则/模板帖 |

**改皮肤**：编辑 `theme-v2.js` 顶部 `:root` 里的 CSS 变量（调色只改一处），然后：

```bash
scp ops/forum/theme-v2.js aliyun:/srv/heuesta/forum/config/
ssh aliyun "docker exec -w /usr/src/app heuesta-forum-forum-1 node /opt/config/theme-v2.js && \
  docker compose -f /opt/heuesta/web/ops/forum/docker-compose.yml --env-file /opt/heuesta/forum.env restart forum"
```

注意：自定义 CSS 走 `meta.configs.setMultiple`（会被 sass 编译成 `renderedCustomCSS` 注入页面），
直接往 DB 写 `customCSS` 字段是**不生效**的。

**加/改版块**：小改动直接在论坛 ACP → 管理 → 版块 操作即可；成批调整改 `categories-v2.js` 后同上执行。
分区（父版块）名不要与任何子版块重名，否则脚本会报 self-as-parent 错误。

**论坛横幅**：图片在 `/srv/heuesta/forum/uploads/system/banner.webp`，直接替换同名文件即可（建议宽 1536px 的 WebP）。

### 8.3 皮肤与版块管理（科协风格化）

论坛外观与版块结构由 `ops/forum/` 下的幂等脚本管理（改完 scp 到 `/srv/heuesta/forum/config/` 后 `docker exec` 执行）：

| 脚本 | 管什么 |
| --- | --- |
| `theme-v2.js` | 全站皮肤 CSS（调色改文件顶部 `:root` 变量）+ 帖子页装饰 JS（楼层号/作者栏/横幅/QQ条） |
| `categories-v2.js` | 6 分区 14 子版块结构（增删版块改 `STRUCTURE` 数组） |
| `seed-v2.js` | 各版块置顶导航帖 |

要点：
- **改 CSS 后必须重启论坛容器**（customCSS 会经 sass 编译为 renderedCustomCSS 注入页面）
- 横幅图片在 `/srv/heuesta/forum/uploads/system/banner.webp`，替换文件即可换横幅
- 日常小调整也可以直接在论坛 ACP → 外观 → Custom Content 里改（和脚本内容一致，注意两边同步）
- 楼层号规则：楼主 / 沙发 / 板凳 / 地板 / #N楼；MCBBS 式「点评」「评分」暂无对应插件，属二期自研范围

## 9. 实验室虚拟机（异地备份 + 内网资料库）

实验室 Windows 主机上跑着一台 VMware Ubuntu 虚拟机（内网 IP `192.168.1.112`，SSH 账号 xiazhiyuan），承担两个角色。
**注意：Windows 主机要保持开机，并建议在 VMware 里把该虚拟机设为随主机自动启动。**

### 9.1 异地备份（已部署，自动运行）

云端每日 03:30 备份后，虚拟机 04:30 自动拉取全部备份到 `~/heuesta-backups/cloud/`，本地保留 30 天（云端只留 7 天）。

- 云端限权账号 `backup-puller`：经 rrsync 只读访问 `/srv/heuesta/backups`，无 shell、无 root
- 查看拉取日志：虚拟机上 `tail ~/heuesta-backups/pull.log`
- 灾难恢复：拿本地最新的 `db-*.sql.gz`（主站）、`forum-db-*.sql.gz`（论坛）、`media-*.tar.gz`、`forum-uploads-*.tar.gz`，
  在新服务器上按第 5 节「恢复」流程导入即可整站重建

### 9.2 内网资料库（Filebrowser，大文件 / 内部资料专用）

- 地址：**http://192.168.1.112:8080**（仅实验室内网可访问，公网不可达 —— 内部资料不出内网）
- 与云端资料站的分工：官网资料站放小体积、可公开或会员级的资料；**大文件（影音、镜像、数据集）和内部敏感资料放内网资料库**
- 磁盘：虚拟机盘 116GB（不够时在 VMware 里扩 vmdk 后 `lvextend` 扩容）
- 管理员账号见交接密码册；给成员开账号：登录 → 设置 → 用户管理 → 新建（可按目录限定权限，如只读）
- 目录结构：培训资料 / 竞赛资料 / 内部文档 / 软件工具 / 影音素材（`/srv/esta-files/`）
- 运维（虚拟机上）：`cd /opt/filebrowser && docker compose restart`；数据库在 `/opt/filebrowser/database`
- 一键重装脚本：[ops/offsite/vm-setup.sh](../ops/offsite/vm-setup.sh) 与 [ops/offsite/vm-fix.sh](../ops/offsite/vm-fix.sh)

## 11. 会员体系 2.0（等级 / 职位 / 勋章）

### 11.1 五级会员等级

单一事实来源是 User.member_level：

| 等级 | 含义 | 权限要点 |
| --- | --- | --- |
| 0 待审核 | 注册后未通过 | 只能看公开内容 |
| 1 报名会员 | 已报名未面试 | 解锁"报名会员级"资料 |
| 2 预备会员 | 过一轮面试 | 解锁"预备级"资料 |
| 3 正式会员 | 过二轮面试 | 解锁正式级资料、论坛「内部事务」版块 |
| 4 干事 | 协会干事 | 进驾驶舱、传资料、审核会员 |
| 5 管理员 | 站点管理 | 站点设置、勋章职位、高级后台 |

- 显示统一为「25届·正式会员」（届别取自入学年份 grade）
- 改等级：驾驶舱 → 会员管理 → 勾选 → 批量按钮（通过审核 / 晋升预备 / 晋升正式 / 设为干事 / 设为管理员）；每次变更自动写 `等级变更记录`（后台可查，换届追溯用）
- 论坛头衔会自动跟随等级变化（晋升后旧徽章自动摘除）

### 11.2 职位与勋章

- 职位（主席/硬件主席/软件主席等）：驾驶舱 → 职位管理，创建后任命给成员；会显示为个人中心和论坛的彩色头衔
- 勋章：驾驶舱 → 勋章管理，创建勋章（emoji 图标）后授予成员，显示在个人中心勋章墙
- 授予/任命需要成员的用户 ID（在会员管理页每行可见 `#ID`）

### 11.3 两个内测开关（驾驶舱 → 站点设置）

- **内测模式**：开启时①全站显示"🐛 内测反馈"浮动按钮；②新注册用户直接给干事级（方便测全部功能）。正式招新前**务必关闭**，否则新人一注册就是干事。
- **注册自动审核**：开启时注册通过邮箱验证即激活为报名会员；关闭则回到"注册→管理员手动审核"。
- 关闭内测模式后，若要批量把测试期的干事降回正式会员，在会员管理里筛选"干事"逐个降级即可。

### 11.4 邮箱验证码（注册/找回密码/验证码登录）

发信走**阿里企业邮箱免费版**（noreply@heuesta.cn，后台 qiye.aliyun.com，postmaster 账号管理，免费 50 个成员邮箱、对外 2000 封/天）。`/opt/heuesta/.env` 中的现行配置：

```bash
EMAIL_HOST=smtp.qiye.aliyun.com
EMAIL_PORT=465
EMAIL_USE_SSL=1
EMAIL_HOST_USER=noreply@heuesta.cn
EMAIL_HOST_PASSWORD=（noreply 账号的密码，改密后同步改这里）
```

注意事项：
- 邮箱 DNS 记录（MX/SPF/CNAME）在 **Cloudflare** 上，别删；
- 新建成员邮箱：postmaster 登录 → 组织与用户 → 新建账户，勾选"开启 POP3/SMTP"；
- 如果换了 noreply 密码，改 .env 后需 `docker compose ... up -d app` 重建容器生效。

改完 `sudo /opt/heuesta/web/ops/deploy.sh` 重启生效。**未配置时验证码不会发邮件，而是打印到 app 容器日志**（`docker compose ... logs app`），仅供应急。

### 11.5 内测反馈管理

- 内测模式开启时，主站每页右下角有「🐛 内测反馈」大按钮（弹窗直接提交），论坛的同款按钮跳到 heuesta.cn/feedback/ 独立页；匿名也可提交（有 30 秒冷却 + 每 IP 每日 30 条限流）
- 所有反馈进数据库：**驾驶舱 → 反馈管理** 统一审阅，逐条「标记已处理」（可写处理备注，提交人在反馈页能看到回复）；删除需管理员
- 侧边栏"反馈管理"红色角标 = 待处理条数

### 11.6 分级资料

上传资料时选"可见等级"（公开 / 报名会员及以上 / 预备 / 正式 / 干事）。用户等级达到门槛才能在列表看到并下载。

### 11.7 论坛分级版块

「内部事务」版块仅正式会员及以上可见。要再加分级版块：论坛 ACP → 版块 → 新建 → 权限里撤销 guests/registered-users 的读权限，只给对应等级组；或参考 `ops/forum/internal-board.js` 批量脚本。

## 12. 公告系统与通知中心（v1.1 新增）

### 12.1 公告（news）

- 入口：主站导航「公告」（/news/），分三个栏目：**通知公告 / 协会动态 / 获奖喜报**
- 发布：**驾驶舱 → 公告管理 → 写公告**（干事及以上）。正文为 Markdown（标题、加粗、列表、表格、图片、链接均可），服务端渲染并做 XSS 消毒（nh3 白名单）
- 正文配图：编辑页「插入图片」按钮上传（JPG/PNG/GIF/WebP，≤10MB），自动插入 Markdown 图片语法；封面图选填，展示在列表卡片和详情页顶部
- 每篇公告可设**可见等级**（同资料站的分级逻辑）、**置顶**、**发布/下架**；「发布时间」设为未来时间即定时发布（到点自动可见）
- 首页 Hero 下方自动展示最新 3 篇（对当前访客可见的）公告；没有公告时该区块整体隐藏
- 删除公告：作者本人或管理员
- 依赖：`Markdown`、`nh3`（已在 requirements.txt，部署时自动安装）

### 12.2 通知中心（notify）

- 登录用户导航栏出现**铃铛**：红点数字 = 未读数；点击展开最近 8 条下拉（移动端直接跳列表页）；全量在 /notify/（支持「未读」筛选、全部标为已读）
- 自动触发的站内通知：
  - **等级变更**（`User.set_level`，即驾驶舱批量晋级、审核通过等都会通知本人，备注一并显示）
  - **勋章授予**（驾驶舱勋章管理授予时）
  - **反馈动态**（管理组回复反馈 / 标记已处理时通知提交人）
- 给新模块加通知：`from notify.services import notify_user, notify_users`，传 `kind`（notify/models.py 的 `Notification.Kind`）、`title`、`body`、`url`（站内相对路径）即可，勿直接 create Notification
- 后台（Admin → 站内通知）可查全部通知流水

## 13. 活动系统与积分（v1.1 新增）

### 13.1 活动（events）

- 入口：主站导航「活动」（/events/），分「即将开始 / 已结束」两个标签，可按类型（培训 / 讲座 / 竞赛宣讲 / 例会 / 其他）筛选
- 发布：**驾驶舱 → 活动管理 → 新建活动**（干事及以上）。字段含类型、Markdown 详情、地点、起止时间、报名截止（留空=开始前一刻）、人数上限（0=不限）、报名等级门槛、签到积分
- 报名：成员在活动详情页一键报名 / 取消；满员、截止、等级不够都会被拦下；报名成功发一条站内通知
- **现场签到**：驾驶舱 → 活动管理 → 对应活动「报名名单」页 → **开启签到**，系统生成 6 位口令并同时给出二维码
  - 投屏二维码：成员手机扫码后自动跳到活动页并填好口令，点「签到」即完成
  - 或口头公布口令，成员在活动页手动输入
  - 未提前报名的人签到会自动建一条「现场参加」记录
  - 签到成功按活动设定的积分自动加分（走 points，见 13.2），并发通知
  - 干事也可在报名名单页对某人「代签到」
- 名单导出：报名名单页「导出 CSV」（含姓名 / 学号 / 学院 / 等级 / 报名时间 / 签到状态，UTF-8 BOM，Excel 直接打开不乱码）
- **活动开场提醒**（可选，需服务器配置）：`send_event_reminders` 管理命令会给「1 小时内开始、已报名、还没提醒过」的成员发站内通知；靠 `heuesta-event-reminder.timer` 每 15 分钟触发（见 4 节部署，deploy.sh 会自动装好 timer）。不装 timer 也不影响报名签到，只是少了自动提醒

### 13.2 积分（points）

- 唯一事实来源是 `PointLog` 流水表，某人总分 = 其所有流水 `delta` 之和；**不要**直接改用户身上的某个总分字段（没有这种字段），一律走 `points.services.award_points()`
- 积分来源目前两种：**活动签到**（自动，按活动 `points_reward`）、**管理调整**（手动）
- 手动加/扣分：驾驶舱 → 会员管理 → 勾选成员 → 「积分调整」，填分值（可负）和说明（如「电赛校赛一等奖 +8」）；会记流水并通知本人
- 展示：个人中心显示总积分入口，/points/ 是本人积分明细，/points/leaderboard/ 是全站积分榜
- 给新来源加分：`from points.services import award_points`，传 `user`、`delta`、`source`（PointLog.Source）、`note`；`delta=0` 会被忽略

## 14. 招新报名（v1.1 新增）

线上化「报名 → 一面 → 二面 → 晋级」流程，直接和会员等级体系咬合。

### 14.1 开一次招新

- 驾驶舱 → **招新管理 → 新建批次**：填批次名（如「2026 秋季招新」）、招新说明（Markdown）、开放/截止时间、勾选「启用」
- 只有「启用 + 在时间窗内」的批次才对外开放；**同一时间建议只启用一个批次**
- 招新开放期间，主站导航会自动多出「招新」入口（`/recruitment/`），关闭后自动隐藏（该状态缓存 5 分钟）

### 14.2 学生报名

- 学生登录后在招新页填表（意向部门 / 特长 / 自我介绍）提交；提交即建报名记录，并自动挂到「报名会员」等级
- 学生可在招新页和个人中心随时看到自己的报名进展；正式会员及以上不再显示报名表

### 14.3 面试与晋级（驾驶舱 → 招新管理）

- 顶部按批次切换、按状态（已报名/一面通过/二面通过/未录取）筛选，可导出 CSV 作面试名单
- 勾选报名者后用批量按钮处理，可选填一条面试备注写入所选记录：
  - **一面通过 → 预备**：报名状态记「一面通过」，同时把本人**晋升为预备会员**
  - **二面通过 → 正式**：晋升为**正式会员**
  - **未录取**：仅记状态并给本人发一条委婉的站内通知，不改等级
  - **重置**：状态回到「已报名」（不降级，如需降级去会员管理）
- 晋级复用 `User.set_level`：自动写等级变更记录、同步论坛头衔组、给本人发站内通知（无邮件，与会员管理的批量晋级行为一致）

## 15. 项目档案库（v1.1 新增）

给课题/作品一个资料归档空间：项目成员协作上传管理文件，其他会员只能看到项目名片。

### 15.1 建项目、指派负责人（驾驶舱 → 项目档案）

- **新建项目**：填名称、方向（硬件/软件/软硬结合/其他）、简介；可直接填「负责人」（用户名或学号），创建后自动指派为负责人
- 列表里可**归档 / 恢复**项目；**删除**项目（含其所有文件）仅管理员可操作
- 干事及以上能在驾驶舱看到并管理所有项目

### 15.2 项目文件与成员（项目页 `/projects/<id>/`）

- 权限分层：
  - **报名会员及以上**：能看到项目档案库的项目名片列表
  - **项目成员 / 干事**：能进入项目、浏览目录树、上传/新建目录/下载/删除文件
  - **负责人 / 干事**：能添加、移除项目成员
- 目录树：支持在任意层级新建子目录、上传文件（可多选）、逐个下载/删除、删除目录（会连同其中文件一并删除）
- 成员管理在项目页右侧：用用户名或学号添加成员，选「成员 / 负责人」角色
- 导航「项目」入口对登录的报名会员及以上显示

### 15.3 文件存储与安全

- 上传的项目文件放在 `media/projects/<项目id>/`，**nginx 已禁止直链**（`/media/projects/` 返回 403）
- 下载走 Django 鉴权后交给 nginx `X-Accel-Redirect`（同资料站机制，大文件不占应用内存）
- 上传大小上限沿用 nginx 的 200MB
- **注意**：新增了 `location /media/projects/ { return 403; }`，deploy.sh 会自动更新 nginx 配置；若手工改过 nginx，记得同步这条

## 10. 安全须知

- `/opt/heuesta/.env` 里是全部密钥，别提交到 git、别发群里
- SSH 私钥（`.pem`）同理
- 数据库端口没有对外映射，只有 app 容器能连
- 会员密码是 Django 标准哈希存储，谁也看不到明文
- 旧静态站在 `legacy-static` 分支，翻车了随时能回滚：nginx 配置改回指向 `/var/www/heuesta.cn/public` 即可
