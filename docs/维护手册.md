# HEU ESTA 官网维护手册

> 写给每一届接手网站的同学。看完这份文档，你应该能独立完成日常维护和二次开发。
> 遇到问题先看「故障排查」一节；再解决不了，把报错信息发给 AI 助手（Cursor / DeepSeek 等），效率最高。

## 1. 整体架构

```text
用户浏览器
   │
   ▼
CDN（缓存静态资源，减轻服务器压力）
   │
   ▼
阿里云服务器（Ubuntu 24.04，2核/1.6G + 2G swap）
   │
   ├─ nginx（宿主机，443 端口，SSL 证书 Let's Encrypt）
   │    ├─ /static/  → /srv/heuesta/static/   （CSS/JS/图片，一年长缓存）
   │    ├─ /media/   → /srv/heuesta/media/    （后台上传的轮播图、头像）
   │    ├─ /media/resources/ → 403            （资料文件禁止直链）
   │    ├─ /protected/ → internal             （Django 鉴权后由 nginx 发文件）
   │    ├─ /learn/   → /srv/heuesta/site/learn/（电子学学习中心，纯静态）
   │    └─ /         → 127.0.0.1:8001         （反向代理到 Django）
   │
   ├─ Docker: heuesta-app（Django + gunicorn，2 worker）
   ├─ Docker: heuesta-db （PostgreSQL 16）
   └─ Docker: hbbs/hbbr  （RustDesk 中继，与网站无关，别动）
```

关键路径速查：

| 位置 | 内容 |
| --- | --- |
| `/opt/heuesta/web` | 代码仓库（git checkout） |
| `/opt/heuesta/.env` | 生产密钥（SECRET_KEY、数据库密码），**不在 git 里，删了就没了** |
| `/srv/heuesta/static` | collectstatic 输出 |
| `/srv/heuesta/media` | 用户上传文件 |
| `/srv/heuesta/pgdata` | 数据库数据 |
| `/srv/heuesta/backups` | 每日自动备份（保留 7 天） |
| `/etc/nginx/sites-available/heuesta.cn` | nginx 站点配置 |

## 2. 本地开发环境

见 README「本地开发」。要点：

- 开发用 SQLite（`app/db.sqlite3`），不用装 Postgres
- `python manage.py runserver` 默认加载 `config.settings.dev`
- 前端零构建链：改 `app/static/` 下的 CSS/JS 刷新即见效
- 改了模型（models.py）之后：`python manage.py makemigrations && python manage.py migrate`
- 提交前跑测试：`python manage.py test accounts files`

## 3. 日常维护任务

### 3.1 审核新会员

1. 打开 `https://heuesta.cn/admin/`，用管理员账号登录
2. 左侧「会员管理 → 成员」，筛选「有效 = 否」的就是待审核用户
3. 勾选 → 顶部动作选「通过审核并设为会员」→ 执行

### 3.2 设置干事 / 管理员

- 干事（可在前台上传资料）：编辑该成员 → 「组」里加入 `干事`
- 管理员（可进后台）：加入 `管理员` 组，并勾选「工作人员状态」
- 完全管理权（能改站点配置）：再勾选「超级用户状态」（谨慎）

### 3.3 换招新视频 / 改 QQ 群号

Admin → 「站点内容 → 站点配置」，改「招新视频 BV 号」「招新 QQ 群号」，保存即生效（首页缓存最多 1 小时后刷新，可重启 app 容器立即生效）。

### 3.4 更新首页轮播图（科协合影等）

Admin → 「站点内容 → 首页轮播图」→ 上传图片（建议先压缩到 500KB 以内，宽 1600px 左右）。
只要数据库里有启用的轮播图，首页就用后台的；一张没有时才显示仓库自带的兜底图 + 留白占位格。

### 3.5 上传资料

干事及以上登录后，资料站页面会出现「+ 上传资料」按钮。「仅会员」的资料非会员无法下载（鉴权在 Django，文件传输在 nginx，大文件也不占应用内存）。

## 4. 发布新版本

```bash
# 本地
git add -A && git commit -m "更新说明" && git push origin main

# 服务器（ssh aliyun）
sudo /opt/heuesta/web/ops/deploy.sh
```

deploy.sh 是幂等的，失败了修复后再跑一次即可。它不会动数据库数据和上传文件。

## 5. 备份与恢复

- 自动备份：`heuesta-backup.timer` 每天 03:30 备份数据库 + media 到 `/srv/heuesta/backups`，保留 7 天
- 手动备份：`sudo /opt/heuesta/web/ops/backup.sh`
- **建议每学期把备份拷一份到本地**：`scp aliyun:/srv/heuesta/backups/db-最新.sql.gz .`

恢复数据库（危险操作，先确认备份文件没问题）：

```bash
cd /opt/heuesta/web
gunzip -c /srv/heuesta/backups/db-XXXX.sql.gz | \
  docker compose -f ops/docker-compose.yml --env-file /opt/heuesta/.env \
  exec -T db psql -U heuesta -d heuesta
```

恢复上传文件：`tar -xzf media-XXXX.tar.gz -C /srv/heuesta`

## 6. 如何做二次开发

### 加一个新页面

1. `app/core/views.py` 加视图函数
2. `app/core/urls.py` 加路由
3. `app/templates/core/` 加模板（`{% extends "base.html" %}` 即自动获得导航栏页脚）
4. 导航栏入口在 `app/templates/base.html`

### 加一个新模块（比如论坛）

1. `python manage.py startapp forum`
2. `config/settings/base.py` 的 `INSTALLED_APPS` 里注册
3. `config/urls.py` 挂路由：`path("forum/", include("forum.urls"))`
4. 权限直接复用 `accounts.roles` 里的 `is_member` / `is_officer`
5. 用户外键写 `settings.AUTH_USER_MODEL`

会员系统、角色组、登录鉴权都是现成的，论坛只需要写帖子/回复的模型和页面。

### 更新学习中心

`learn/electronics/` 是纯静态文件，直接改 HTML，push 后 deploy 即可。新增章节记得同步 `manifest.json` 与 `index.html`。

## 7. 故障排查

| 症状 | 排查 |
| --- | --- |
| 网站 502 | `docker ps` 看 app 容器是否在跑；`docker compose ... logs app` 看报错 |
| 部署后页面没变 | 浏览器强刷（Ctrl+F5）；CDN 有缓存的话去 CDN 控制台刷新；确认 deploy.sh 跑完没报错 |
| 上传文件失败 | 看 nginx `client_max_body_size`（现 200m）与磁盘空间 `df -h` |
| B 站数据不显示 | 正常降级，B 站接口偶尔抽风，1 小时缓存过期后自动恢复；不影响其他内容 |
| 内存吃紧 | `free -h`；app 容器限 450m、db 限 300m；已配 2G swap 兜底 |
| 证书过期 | certbot 自动续期：`systemctl list-timers | grep certbot`；手动 `certbot renew` |
| 忘了管理员密码 | `docker compose ... exec app python manage.py changepassword <用户名>` |

## 8. 安全须知

- `/opt/heuesta/.env` 里是全部密钥，别提交到 git、别发群里
- SSH 私钥（`.pem`）同理
- 数据库端口没有对外映射，只有 app 容器能连
- 会员密码是 Django 标准哈希存储，谁也看不到明文
- 旧静态站在 `legacy-static` 分支，翻车了随时能回滚：nginx 配置改回指向 `/var/www/heuesta.cn/public` 即可
